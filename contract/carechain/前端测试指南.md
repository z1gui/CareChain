## 启动前端

```bash
# 进入前端目录
cd app

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

启动后浏览器访问 `http://localhost:5173`，会看到 **CareChain 测试网面板**，包含三个操作面板：

| 面板 | 说明 |
|------|------|
| 🏥 初始化 Facility | 创建养老设施（管理员操作） |
| 🛏️ 创建 Bed Class | 创建床位类型（管理员操作） |
| 🪙 Mint BedRight NFT | 铸造 NFT（用户操作） |

> 测试环境下三个面板在同一页面，方便快速走完全流程。

## 测试流程

### 第一步：连接钱包

1. 点击页面右上角的 **「Select Wallet」** 按钮
2. 选择 **Phantom**
3. 在弹出的 Phantom 窗口中点击「连接」
4. 连接成功后按钮会显示你的钱包地址缩写

> ⚠️ 确保 Phantom 已切换到 Devnet，且钱包有足够的 SOL（至少 0.5 SOL 用于交易手续费和账户租金）。

---

### 第二步：准备测试 USDC

由于 Devnet 上没有真正的 USDC，需要自行创建一个**模拟 USDC 代币**。

#### 2.1 创建模拟 USDC Mint

在终端中执行（需要先进入 WSL Ubuntu 环境）：

```bash
# 创建一个 6 位精度的代币（模拟 USDC）
spl-token create-token --decimals 6
```

命令输出会给你一个 **Token Mint 地址**，记录下来，例如：
```
Creating token AbC123...xYz
```

#### 2.2 创建你的代币账户 (ATA)

```bash
spl-token create-account <上一步的Mint地址>
```

#### 2.3 给自己铸造测试 USDC

```bash
# 铸造 10000 个测试 USDC（含 6 位精度）
spl-token mint <Mint地址> 10000
```

#### 2.4 创建金库的代币账户

金库需要一个独立的 ATA 来接收铸造 NFT 的 USDC 收入。你可以用自己的钱包地址作为金库（测试用），也可以创建新地址：

```bash
# 查看你的 ATA 地址（用自己的钱包作为金库时）
spl-token accounts
```

> **记录以下两个地址，后续步骤会用到：**
> 1. **模拟 USDC Mint 地址** — 填入铸造 NFT 面板
> 2. **金库 ATA 地址** — 填入初始化设施面板

---

### 第三步：初始化养老设施

在 **「🏥 初始化 Facility」** 面板中：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 设施 ID | `FAC-001` | 设施唯一标识，可保持默认 |
| 设施名称 | `Sunny Care Home` | 设施名称，可自定义 |
| 城市 | `Shanghai` | 可自定义 |
| 总床位数 | `100` | 设施总床位 |
| 金库 USDC ATA 地址 | （空） | **必填！** 填入上一步创建的金库 ATA 地址 |

填写完成后点击 **「执行 Initialize Facility」**，Phantom 会弹出签名确认。

✅ 成功时按钮会显示 `成功! TX: xxxxxxxx...`

> ⚠️ 同一个 `facility_id` 只能初始化一次。如果提示「账户已存在」，换一个不同的设施 ID（如 `FAC-002`）。

---

### 第四步：创建床位类型

在 **「🛏️ 创建 Bed Class」** 面板中：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 所属设施 ID | `FAC-001` | **必须与第三步的设施 ID 一致** |
| 床位分类 ID | `BED-S1` | 床位类型唯一标识 |
| 房间类型 | `Standard Single` | 房间描述 |
| 护理等级 | `Independent Living` | 护理级别 |
| 售价 (含精度) | `500000000` | 500 USDC（6 位小数 × 500） |
| 年化收益 (BPS) | `650` | 6.5%（basis points） |

> 注意：`totalSupply` 和 `privilegeLevel` 使用代码中的默认值（50 和 P1）。

填写完成后点击 **「执行 Create Bed Class」**，等待 Phantom 确认。

✅ 成功时按钮会显示 `成功! TX: xxxxxxxx...`

> ⚠️ 执行此操作的钱包**必须是初始化设施时使用的同一个钱包**（即 authority），否则会报 `has_one` 约束错误。

---

### 第五步：铸造 BedRight NFT

在 **「🪙 Mint BedRight NFT」** 面板中：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 设施 ID | `FAC-001` | 与前两步一致 |
| 床位分类 ID | `BED-S1` | 与第四步一致 |
| USDC Mint 地址 | （空） | **必填！** 填入第二步创建的模拟 USDC Mint 地址 |

填写完成后点击 **「执行 Mint NFT」**，Phantom 会弹出签名确认。

✅ 成功时会显示 `成功! NFT Mint: <地址> TX: xxxxxxxx`

> 铸造成功后：
> - 你的钱包会收到 1 枚 BedRight NFT
> - 你的 USDC 余额会减少对应的 `price_usdc` 金额
> - 金库 ATA 会收到对应的 USDC
> - 链上的 `minted_supply` 和 `sold_nfts` 计数器会 +1

## 验证结果

### 在 Solana Explorer 查看交易

将交易签名（TX hash）粘贴到 [Solana Explorer](https://explorer.solana.com/?cluster=devnet) 查看交易详情。

### 用 CLI 验证链上状态

```bash
# 查看你的 USDC 余额变化
spl-token balance <USDC_Mint地址>

# 查看金库余额
spl-token balance <USDC_Mint地址> --owner <金库钱包地址>

# 查看你持有的 NFT
spl-token accounts --owner <你的钱包地址>
```

### 在 Phantom 钱包查看 NFT

铸造成功后，打开 Phantom 钱包的「收藏品」标签页，应该能看到一枚名为 **CareChain BedRight** 的 NFT，符号为 **BED**。
